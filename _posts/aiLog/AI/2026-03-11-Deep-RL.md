# Deep RL 등장 배경

{: .info-box}
Deep RL 의 각 개념은 독립적으로 존재하는 것이 아니라, 앞선 방법의 한계를 해결하기 위해 순서대로 등장했다. 전체 흐름을 먼저 파악하면 각 개념이 왜 필요한지 자연스럽게 이해된다.

{: .center}
```
기존 강화학습 (Tabular RL)
한계: 상태 폭발(State Explosion) 문제. 바둑이나 고해상도 게임처럼 상태가 너무 많으면 테이블(Matrix)에 다 저장할 수 없음.
↓
근사 (Approximation)
아이디어: 값을 일일이 저장하는 대신, 입력($s$)을 넣으면 값(V)이 나오는 함수(f)를 만들자.
↓
신경망 (Neural Network)
구현: 복잡한 비선형 함수를 가장 잘 모사하는 인공신경망을 근사 함수로 채택.
↓
가치 기반 RL (DGN 등)
메커니즘: 행동 가치(Q)를 신경망으로 근사하여 가장 높은 Q값을 가진 행동을 선택(max).
한계: 연속 행동 공간(Continuous Action Space) 적용 불가. 무한한 선택지 중 $max$값을 일일이 계산하는 것은 불가능함.
↓
정책 기반 RL (Policy Gradient / REINFORCE)
아이디어: 점수를 매기지 말고, 신경망이 직접 행동의 확률(𝝅)을 출력하게 하자.
한계: 에피소드 전체 보상에 의존하므로 분산(Variance)이 너무 커서 학습이 매우 불안정함.
↓
PPO (Proximal Policy Optimization)
최종 진화: Actor-Critic 구조를 유지하되, Clipping(변화폭 제한) 기법을 도입하여 "안전한 범위 내에서만" 업데이트하도록 설계. 현재 가장 대중적이고 안정적인 표준 알고리즘.
```

# 기존 강화학습의 한계 - 왜 Deep 이 필요한가?

{: .info-box}
Monte Carlo, TD Learning, Q-Learning 같은 기존 강화학습은 각 상태의 가치를 테이블(표)에서 직접 저장하는 Tabular 방식을 사용한다. 이 방식은 상태가 적을 때는 잘 작동하지만, 현실 문제에서는 결정적인 한계에 부딪힌다.

## 현실 문제에서의 상태 수

|문제|상태 수|이유|
|--------|----------|-----------------------|
|**체스**| ≈ $10^43$ | 8x8 보드, 말 조합 |
|**바둑**| ≈ $10^170$ | 19x19, 각 칸 3가지 |
|**Atari 게임**| 천문학적 | 84x84 픽셀 이미지 = 각 픽셀 값 조합  |
|**자율주행**| 사실상 무한 | 위치・속도・신호 등 연속값 |

> "바둑에서 $10^170$ 개의 상태를 모두 테이블에 저장하려면               
> 우주의 원자 수(≈$10^80$)보다 많은 메모리가 필요하다."

이처럼 상태가 너무 많거나 연속적인 경우, 테이블 방식은 근본적으로 불가능하다.           
그래서 등장한 개념이 **근사(Approximation)** 이다.

# 근사 (Approximation) - 저장 대신 계산

{: .info-box}
근사의 핵심 아이디어는 간단하다.                    
모든 상태의 값을 일일이 저장하는 대신, 그 값을 계산해주는 함수를 만들자는 것이다.

{: .star-list}
- **테이블 방식:** 상태 → [테이블에서 값을 찾아옴]
- **함수 근사:** 상태 → [함수에 넣어서 값을 계산함]
- **저장하는 것:** 
  - 테이블 = 모든 상태의 값
  - 함수 = 파라미터(가중치) 몇 개

$$V(s) = f(s)$$

> 파라미터 몇 개만 저장하면 상태가 아무리 많아도 대응할 수 있다.            
> 그리고 경험을 통해 바뀌는 것도 상태 값이 아니라 파라미터이다.         
> 즉, 경험이 파라미터 안에 압축되어 저장된다.

## 함수 근사 (Functional Approximation) 예시

{: .info-box}
가장 단순한 형태 - 상태가 숫자 하나일 때

$$V(s) = w x s$$

|기호|의미|
|---|--------|
|**s**| 상태 |
|**w**| 학습되는 파라미터 - 경험을 통해 조금씩 바뀌는 값 |

```
w = 0.5

def V(s):
    return w * s    # 상태 값이 들어올 때마다 계산으로 가치 결정
```

> 이렇게 하면 상태 값이 들어올 때마다 계산으로 가치가 결정된다. 테이블처럼 값을 저장하는 것이 아니라 함수를 이용해 계산하는 방식이다.

## 함수 근사에서 저장되는 것

{: .info-box}
함수 자체에 값을 저장하는 것이 아니라, 함수의 파라미터(parameter) 가 학습된다.

$$V(s) = w_1x_1 + w_2x_2$$

여기서 $w_1$ 과 $w_2$ 가 바로 학습되는 값이다. 강화학습에서 경험을 통해 바뀌는 것은 상태 값이 아니라 함수의 파라미터이다. 즉, 경험이 파라미터 안에 압축되어 저장된다.

{: .star-list}
- **테이블 방식**: 상태 1,000개 → 1,000개 값을 메모리에 저장                
- **함수 근사**: 상태 1,000개 → 파라미터 $w_1, w_2$ 2 개만 저장

> 상태가 아무리 많아도 파라미터 수는 고정 → 메모리 효율적

## 일반화(Generalization) - 처음 보는 상태도 처리 가능

{: .info-box}
테이블 방식은 학습 중에 한 번도 방문하지 않은 상태가 나타나면 아무런 정보가 없다. 하지만 함수 근사는 다르다.

|방식|처음 보는 상태 대응|
|-----|---------------------|
|**테이블 방식**| 정보 없음 - 처리 불가 |
|**함수 근사**| 비슷한 상태에서 학습한 패턴으로 합리적 예측 가능 |

함수 근사는 여러 상태에서 공통적으로 나타나는 패턴을 학습하기 때문에, 처음 보는 상태도 비슷한 특성을 가지면 합리적인 가치를 예측할 수 있다. 이것이 **일반화** 이며, 상태 공간이 큰 강화학습에서 핵심 능력이다.

## 피팅(Fitting) - 어떤 함수든 사용 가능

{: .info-box}
**피팅(Fitting)은 데이터의 패턴을 가장 잘 설명하도록 함수의 파라미터를 조절하는 과정** 이다. 여러 데이터 점이 있을 때 그 점들의 패턴을 가장 잘 따라가는 직선이나 곡선을 찾는 것이 피팅이다.

|**지수・로그 함수**| 특징 성장/감소 패턴에 사용 |
|**신경망 (Neural Net)**| 매우 복잡한 비선형 패턴. 가장 강력한 모델 |

### Underfitting (과소적합)

모델이 너무 단순하거나 학습이 충분히 이루어지지 않아 데이터의 패턴을 제대로 반영하지 못하는 경우이다. 실제 데이터가 곡선 형태인데 직선으로 설명하려 하면 전바적인 경향조차 맞추지 못한다. 학습 데이터에서도 오차가 크게 나타나며 예측 성능이 전반적으로 낮아진다.

### Overfitting (과대적합)

모델이 학습 데이터에 너무 과도하게 맞춰진 상태이다. 데이터의 일반적인 패턴뿐 아니라 우연히 포함된 노이즈나 세부 변동까지 모두 학습해버린다. 학습 데이터에서는 정확하지만 새로운 데이터에서는 성능이 크게 떨어진다. (예: Degree 16)

> 즉,           
> - Underfitting: 모델이 너무 단순 → 학습/테스트 모두 성능 낮음
> - Overfitting: 노이즈까지 학습 → 학습 성능은 높지만 새 데이터에서 낮음
> - 적절한 모델: 데이터의 본질적 패턴만 학습 → 새로운 데이터에서도 좋은 성능

# 신경망 (Neural Network) - 가장 강력한 함수 근사기

선형 함수나 단순 다항식으로는 바둑, Atari 게임, 자율주행처럼 극도로 복잡한 상태 패턴을 표현할 수 없습니다. 그래서 강화학습은 신경망을 함수 근사기로 채택했고, 이것이 Deep RL의 출발점이다.

> "신경망은 단순히 함수 근사기 중 하나가 아니라             
> 어떤 복잡한 함수도 표현할 수 있는 범용 근사기이다."

## 강화학습에서 신경망의 역할

{: .info-box}
신경망은 상태(State)를 입력받아 가치 또는 정책을 출력하는 함수 역할을 한다.

$$State → Neural Network → Output$$

|출력 유형|의미 및 사용처|
|-----|----------------------|
|**상태 가치 V(s)**| 현재 상태가 얼마나 좋은가? → Critic, TD Learning |
|**행동 가치 Q(s,a)**| 이 상태에서 각 행동이 얼마나 좋은가? → DQN, Q-Learning |
|**정책 𝝅(a⎮s)**| 이 상태에서 각 행동을 선택할 확률 → Policy Gradient, Actor |

## 신경망의 학습 흐름

{: .center}
```
상태 s 관찰
↓
신경망 → 가치 또는 정책 계산
↓
행동 선택 → 환경에서 실행
↓
보상 r, 다음 상태 s' 수신
↓
경험 (s, a, r, s')으로 가중치 업데이트
↓
더 좋은 예측을 하는 신경망으로 성장
```

> 신경망이 학습한다는 말은, 경험을 통해 가중치(파라미터)가 조금씩 바뀌면서 더 정확한 가치 예측이나 더 나은 행동 선택을 하게 된다는 뜻이다.

# 가치 기반 강화학습 (Value-Based RL)

{: .info-box}
신경망으로 무엇을 근사할 것인가? 첫 번째 선택지는 **행동 가치 함수 Q(s,a)** 이다.           
신경망이 Q 값을 계산하고,           
에이전트는 Q 값이 가장 큰 행동을 선택한다.          
이것이 가치 기반 방법의 핵심이다.

## 가치 기반의 핵심 아이디어

{: .center}
```
현재 상태 s 관찰
↓
신경망으로 모든 행동의 Q(s,a) 계산
↓
$a = argmax_aQ(s,a)$ ← Q 값이 가장 큰 행동 선택
```

$$Q(s, a): 상태 s에서 행동 a를 선택했을 때 기대되는 누적 보상$$
$$a = argmax_aQ(s,a): 상태 s에서 가능한 행동 중 Q값이 가장 큰 행동$$


## DQN - 신경망과 Q-Learning 의 결합

{: .info-box}
DQN(Deep Q-Network)은 기존 Q-Leraning의 Q 테이블을 신경망으로 대체한 알고리즘이다.
Atari 게임의 화면 픽셀을 직접 일력받아 Q 값을 출력한다.

{: .star-list}
- **Q-Learning (기존):** Q 테이블에 Q(s,a) 값 저장
- **DQN (Deep RL):** 신경망이 Q(s,a) 값을 계산
- **입력**: 84x84 픽셀 화면 → 신경망 → 출력: 각 행동의 Q 값

## 대표 알고리즘

{: .star-list}
- **Q-Learning:** 테이블 기반. 다음 상태의 max Q값으로 업데이트 (Off-policy)
- **SARSA:** 현재 정책 기반으로 학습. Q-Learning보다 보수적 (On-policy)
- **DQN:** 신경망으로 Q 근사. Replay Buffer, Target Network 사용
- **Double DQN:** Q값 과대추정 문제 완화
- **Dueling DQN:** 상대 가치 V(s)와 행동 이점 A(s,a)를 분리 학습

## 가치 기반의 한계 - 정책 기반이 등장하는 이유

{: .info-box}
가치 기반은 강력하지만 한 가지 근본적인 한계가 있다.

### 한계

{: .star-list}
- **이산 행동만 가능:** Q(s,a)를 계산하려면 행동의 수가 유한해야 함
- **연속 행동 불가:** 핸들 각도 0.0~1.0 처럼 무한한 행동에는 argmax 계산 불가
- **행동 수 폭발:** 행동이 수천 개 이상이면 Q 값 계산 비용이 폭발적으로 증가

### 예: 자동차 핸들 각도를 -30° ~ 30°로 제어하는 문제

- 가치 기반: 0.1° 단위로 601 개의 이산 행동 → Q값 601 개 계산 필요
- 정책 기반: 직접 -15.3° 라는 연속값을 출력 → 자연스럽게 처리

> "행동이 연속적이거나 너무 많으면 Q 값을 일일이 계산하는 것 자체가 불가능해진다.               
> 그래서 행동의 가치를 계산하는 대신,           
> 어떤 행동을 할지 직접 결정하는 함수를 학습하는 방법이 필요해졌다."

# 정책 기반 강화학습 (Policy-Based RL)

{: .info-box}
가치 기반의 한계를 극복하기 위해 등장한 방법이다. **행동의 가치를 계산하는 것이 아니라, 상태를 보고 어떤 행동을 할지 확률을 직접 출력하는 함수(정책)를 학습** 한다.

## 정책(Policy)이란?

{: .info-box}
정책은 상태가 주어졌을 때 어떤 행동을 선택할 것인지 결정하는 규칙이다. 두 가지 표현 방식이 있다.

|수식|의미|설명|흐름|
|-----|-------|--------------------------|-------------------------|
|**π(a∣s)**| 정책 그 자체 | 상태 s 에서 행동 a 를 선택할 확률. 아직 어떻게 구현할 지 정해지지 않은 추상적 표현 | 상태 → 정책 → 행동 확률 → 행동 선택 |
|**π_θ(a|s)**| 파라미터화된 정책 | θ(신경망 가중치)로 표현된 정책. θ를 조정하면서 학습이 가능한 구체적 표현 | state → neural network → action probability |

$$π(a∣s) → π_θ(a|s) θ: 정책의 파라미터(신경망 가중치)$$

π(a∣s) 는 개념이고, π_θ(a∣s) 는 그것을 신경망으로 구현한 것이다. θ가 있어야 경험으로 파라미터를 업데이트하며 학습할 수 있다.

### 정책 신경망의 출력 예시

{: .center}
```
state → neural network(θ) → [0.1,   0.6,    0.3]
                            ↑       ↑       ↑
                        행동 1 10% 행동 2 60% 행동 3 30%
```

### 연속 행동의 경우

{: .center}
```
state → neural network(θ) → -15.3 (핸들 각도 직접 출력)
```

## 정책 기반의 학습 원리

{: .info box}
좋은 결과를 가져온 행동의 선택 활률은 높이고,               
나쁜 결과를 가져온 행동의 확률은 낮추는 방향으로 파라미터를 업데이트 한다.

|경험|업데이트 방향|
|-------|--------------------|
|**행동 오른쪽, 보상 +10**| 오른쪽 행동의 확률 증가 |
|**행동 왼쪽, 보상 -5**| 왼쪽 행동의 확률 감소 |

## 정책 기반이 필요한 이유 요약

{: .star-list}
- **연속 행동 공간:** 핸들 각도・로봇 팔 각도처럼 연속적인 값을 직접 출력
- **행동 수가 매우 많음:** Q 값을 모두 계산하지 않아도 됨
- **확률적 정책 필요:** 탐험(exploration)에 자연스럽게 유리

## 대표 알고리즘

{: .star-list}
- **Policy Gradient:** 가장 기본. 정책 파라미터를 직접 업데이트
- **REINFORCE:** Monte Carlo 기반 Policy Gradient
- **Actor-Critic:** 정책(Actor) + 가치 함수(Critic) 동시 학습
- **PPO:** 현재 가장 많이 사용. 안정적 업데이트

# 가치 기반 vs 정책 기반

|항목|가치 기반|정책 기반|
|------|-----------------|------------------|
|**학습 대상**| Q(s,a) - 행동의 점수 | $\pi(a⎮s)$ - 행동의 확률 |
|**행동 선택**| argmax Q(s,a) | 확률 분포에서 샘플링 |
|**이산 행동**| 강점 | 가능 |
|**연속 행동**| 어려움 | 강점 |
|**샘플 효율**| 높음 | 낮음 (많은 경험 필요) |
|**학습 안정성**| 비교적 안정적 | 파라미터 조정 중요 |
|**대표 사용처**| Atari 게임, 보드게임 | 로봇 제어, 자율주행 |
