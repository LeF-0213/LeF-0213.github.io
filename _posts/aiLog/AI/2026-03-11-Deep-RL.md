# Deep RL 등장 배경

{: .info-box}
Deep RL 의 각 개념은 독립적으로 존재하는 것이 아니라, 앞선 방법의 한계를 해결하기 위해 순서대로 등장했다. 전체 흐름을 먼저 파악하면 각 개념이 왜 필요한지 자연스럽게 이해된다.

<center>

```
기존 강화학습 (Tabular RL)
한계: 상태 폭발(State Explosion) 문제. 바둑이나 고해상도 게임처럼 상태가 너무 많으면 테이블(Matrix)에 다 저장할 수 없음.
↓
근사 (Approximation)
아이디어: 값을 일일이 저장하는 대신, 입력($s$)을 넣으면 값(V)이 나오는 함수(f)를 만들자.
↓
신경망 (Neural Network)
구현: 복잡한 비선형 함수를 가장 잘 모사하는 인공신경망을 근사 함수로 채택.
↓
가치 기반 RL (DGN 등)
메커니즘: 행동 가치(Q)를 신경망으로 근사하여 가장 높은 Q값을 가진 행동을 선택(max).
한계: 연속 행동 공간(Continuous Action Space) 적용 불가. 무한한 선택지 중 $max$값을 일일이 계산하는 것은 불가능함.
↓
정책 기반 RL (Policy Gradient / REINFORCE)
아이디어: 점수를 매기지 말고, 신경망이 직접 행동의 확률(𝝅)을 출력하게 하자.
한계: 에피소드 전체 보상에 의존하므로 분산(Variance)이 너무 커서 학습이 매우 불안정함.
↓
PPO (Proximal Policy Optimization)
최종 진화: Actor-Critic 구조를 유지하되, Clipping(변화폭 제한) 기법을 도입하여 "안전한 범위 내에서만" 업데이트하도록 설계. 현재 가장 대중적이고 안정적인 표준 알고리즘.
```
</center>

# 기존 강화학습의 한계 - 왜 Deep 이 필요한가?

{: .info-box}
Monte Carlo, TD Learning, Q-Learning 같은 기존 강화학습은 각 상태의 가치를 테이블(표)에서 직접 저장하는 Tabular 방식을 사용한다. 이 방식은 상태가 적을 때는 잘 작동하지만, 현실 문제에서는 결정적인 한계에 부딪힌다.

## 현실 문제에서의 상태 수

|문제|상태 수|이유|
|--------|----------|-----------------------|
|**체스**| ≈ $10^43$ | 8x8 보드, 말 조합 |
|**바둑**| ≈ $10^170$ | 19x19, 각 칸 3가지 |
|**Atari 게임**| 천문학적 | 84x84 픽셀 이미지 = 각 픽셀 값 조합  |
|**자율주행**| 사실상 무한 | 위치・속도・신호 등 연속값 |

> "바둑에서 $10^170$ 개의 상태를 모두 테이블에 저장하려면               
> 우주의 원자 수(≈$10^80$)보다 많은 메모리가 필요하다."

이처럼 상태가 너무 많거나 연속적인 경우, 테이블 방식은 근본적으로 불가능하다.           
그래서 등장한 개념이 **근사(Approximation)** 이다.

# 근사 (Approximation) - 저장 대신 계산

{: .info-box}
근사의 핵심 아이디어는 간단하다.                    
모든 상태의 값을 일일이 저장하는 대신, 그 값을 계산해주는 함수를 만들자는 것이다.

{: .star-list}
- **테이블 방식:** 상태 → [테이블에서 값을 찾아옴]
- **함수 근사:** 상태 → [함수에 넣어서 값을 계산함]
- **저장하는 것:** 
  - 테이블 = 모든 상태의 값
  - 함수 = 파라미터(가중치) 몇 개
