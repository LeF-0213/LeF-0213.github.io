# MDP를 아는 경우 vs 모르는 경우

## MDP 를 아는 경우

{: .info-box}
MDP를 안다는 것은 환경이 어떻게 움직이는지 완전히 알고 있다는 뜻이다. 이 경우 수식으로 직접 Value를 계산할 수 있다.

{: .star-list}
- **상태 S** - 가능한 모든 상태
- **행동 A** - 가능한 모든 행동
- **상태 전이 확률 $P(s'⎮s,a)$** - 행동 후 다음 상태로 이동할 확률
- **보상 R(s, a, s')**  - 각 전이에서 받는 보상

> 벨만 기대 방정식으로 V(s)를 직접 계산 가능

## MDP 를 모르는 경우 (현실)

{: .info-box}
현실에서는 대부분 환경의 상태 전이 확률 $P(s'⎮s,a)$ 를 알 수 없습니다.

- 게임 - 상대방 행동이 예측 불가
- 로봇 - 물리적 마찰, 노이즈 존재
- 자율 주행 - 다른 차량・보행자 예측 불가
- 주식 시장 - 가격 변동 확률 미지수

> P(s'|s,a)를 알 수 없으므로 수식 계산 불가
> → 실제로 행동하면서 얻은 경험으로 Value를 추정해야 함

# 몬테카를로 방법론 (Monte Carlo Method)

몬테카를로 방법론은 확률적 샘플링(무작위 실험)을 반복하여 어떤 값의 근사해나 평균적인 결과를 계산하는 방법이다. 수학적으로 정확한 계산이 어렵거나 복잡할 때, 실제로 많은 무작위 실험을 수행하고 그 결과의 평균을 이용해 답을 추정한다.

> **이름의 유래**                   
> 카지노 도시 MonteCarlo                
> 주사위 ・ 룰렛처럼 확률과 무작위성(randomness)을 활용하는 방식이기 때문이다.

## 핵심 아이디어

|단계|행동|의미|
|----|----------|----------|
| 1️⃣ | 무작위로 실험 | 환경과 상호작용하여 에피소드 실행 |
| 2️⃣ | 아주 많이 반복 | 같은 상태를 다양한 경로로 수없이 방문 |
| 3️⃣ | 결과의 평균 계산 | 평균값이 실제 가치에 수렴 |

> **많은 랜덤 실험 → 평균값 → 실제 값에 가까워짐**

## 장점과 단점

### 장점

- **환경 모델(P, R)이 필요 없음**
- 구현이 단순
- 직관적이고 이해하기 쉬움

### 단점

- **에피소드 종료 후에만 업데이트 가능**
- 긴 에피소드에서 학습이 느림
- **분산이 높아 불안정** 할 수 있음

## 에피소드 종료가 왜 필요한가?

Monte Carlo Learning 에서 상태의 가치를 계산할 때, 그 상태 이후에 얻는 전체 누적 보상(Retrun)을 사용한다. 

어떤 상태에서 시작했을 때 결국 승리(+1)로 끝나는지, 패배(-1)로 끝나는지, 총 보상이 얼마인지는 에피소드가 끝나기 전까지는 확정되지 않는다.

> ### 업데이트 흐름
> 에피소드 실행 완료
>       ↓
> 전체 경로의 누적 보상(Return) 확정
>       ↓
> 역방향으로 각 상태의 Return 계산
>       ↓
> 상태 가치 V(s) 업데이트

# Monte Carlo Learning

{: .info-box}
Monte Carlo Learning 은 에이전트가 여러 번 에피소드를 실행하고, 얻은 총 보상(Return)의 평균으로 상태 가치(Value)를 추정하는 강화학습 방법이다.

{: .star-list}
- **핵심 아이디어:** 같은 상태를 여러 번 방문한 결과의 평균 → 상태 가치 수렴
- **특징:** 모델 프리(Model-Free) - 환경의 P, R을 몰라도 학습 가능
- **제약:** 반드시 에피소드가 종료되어야 업데이트 가능

## Value 업데이트 수식

$$V(s) ← V(s) + \alpha \times  (G - V(s))$$

|기호|의미|
|-----|----------------------|
|**V(s)**| 현재 추정된 상태 s 의 가치|
|**G**| 실제 에피소드에서 얻은 누적 보상(Return) |
|**$\alpha$**| 학습률 - 얼마나 빠르게 업데이트할지 결정 (예: 0.001) |
|**G-V(s)**| 예측 오차 - 실제 Return 과 현재 추정값의 차이 |

> **학습률($\alpha$)**           
> $\alpha$ 가 크면 → 빠르게 학습하지만 불안정
> $\alpha$ 가 작으면 → 안정적이지만 느리게 수렴

# 실전 예제 - GridWorld (4 x 4)

{: .info-box}
4x4 격자 안에서 에이전트가 무작위로 움직이며 Monte Carlo 방식으로 각 칸의 상태 가치를 추정한다.

## 환경 설정

|항목|내용|
|-------|-------------------|
| **격자 크기** | 4 x 4 (총 16개 상태) |
| **시작 위치** | (0, 0) - 좌상단 |
| **목표 위치** | (3, 3) - 우하단, 도착하면 에피소드 종료 |
| **이동 방향** | 상・하・좌・우 4방향 무작위 선택 |
| **벽 처리** | 격자 밖으로 이동하려 하면 제자리 유지 |
| **보상 (R)** | 이동할 때마다 -1 (목표에 빨리 도달할수록 유리) |
| **할인율 γ** | γ=1.0 (미래 보상 전부 반영) |
| **학습률 $\alpha$** | $\alpha$=0.001 |
| **에피소드 수** | 50,000 번 반복 |

### 🗺️ GridWorld 격자

```
(0,0) (0,1) (0,2) (0,3)
(1,0) (1,1) (1,2) (1,3)
(2,0) (2,1) (2,2) (2,3)
(3,0) (3,1) (3,2) (3,3)  ← 목표 지점
```

## 💻 코드 구조

### 환경 - GridWorld 클래스

{: .info-box}
격자판, 위치 이동, 종로 여부 판단을 담당한다.

```python
class GridWorld():
    def __init__(self):
        self.x = 0
        self.y = 0

    def step(self, a):
        if a == 0:
            self.move_left()
        elif a == 1:
            self.move_up()
        elif a == 2:
            self.move_right()
        elif a == 3:
            self.move_down()

        reward = -1
        done = self.is_done()
        return (self.x, self.y), reward, done

    def is_done(self):
        return self.x == 3 and self.y == 3

    def reset(self):
        self.x, self.y = 0, 0
        return (self.x, self.y)
```

### Agent - Agent 클래스

{: .info-box}
4방향 중 하나를 완전 무작위로 선택한다.

```python
class Agent():
    def select_action(self):
        coin = random.random()
        if coin < 0.25:
            return 0
        elif coin < 0.50:
            return 1
        elif coin < 0.75:
            return 2
        else:
            return 3
```

### 학습 - main() 함수

{: .info-box}
50,000 번 에피소드를 반복하며 각 상태의 가치를 업데이트한다.

```python
def main():
    env = GridWorld()
    agent = Agent()
    data = [[0 for _ in range(4)] for _ in range(4)]
    alpha = 0.001
    gamma = 1.0

    for k in range(50000):
        done = False
        history = []

        # 에피소드 실행 - 목표 도달까지 계속 이동
        while not done:
            action = agent.select_action()
            (x, y), reward, done = env.step(action)
            history.append((x, y, reward))

        env.reset()
        cum_reward = 0

        # 역방향으로 Return 계산 후 V(s) 업데이트
        for transition in history[::-1]:
            x, y, reward = transition
            cum_reward = reward + gamma * cum_reward
            data[x][y] = data[x][y] + alpha * (cum_reward - data[x][y])
```

## 역방향 Return 계산 - 왜 거꾸로 더하나?
{: .info-box}
Return(Gt)의 정의는 t 시점 이후에 받을 모든 보상의 합이다. 이를 앞에서부터 계산하면 각 상태마다 에피소드 끝까지 보상을 전부 더해야 한다.

### 앞에서부터 계산할 경우 (비효율적)

```
G(t=0) = r0 + r1 + r2 + ... + rN ← N번 덧셈
G(t=0) =    + r1 + r2 + ... + rN ← N-1번 덧셈
G(t=0) =         + r2 + ... + rN ← N-2번 덧셈
...
```

> 전체 계산량 $O(N^2)$: 에피소드가 길어질수록 폭발적으로 증가


### 뒤에서부터 계산할 경우 (효율적):

{: .info-box}
뒤에서부터 계산하면 점화식을 사용해 이전 결과를 재사용할 수 있다.

$$Gt = Rt+1 + 𝞬 ・ Gt+1$$

```
G(t=N) = 0                          ← 종료 시점, 미래 없음   
G(t=N-1) = r(N-1) + γ x G(t=N)      ← G(N) 재사용
G(t=N-2) = r(N-1) + γ x G(t=N-1)      ← G(N-1) 재사용
...
G(t=0) = r(0) + γ x G(t=1)      ← G(1) 재사용
```

> 전체 계산량 O(N): 각 스텝을 딱 한 번씩만 처리

> - 우하단 영역: 목표에 가까워 상대적으로 높은 가치

## cum_reward 계산 예시

- 에피소드 예시: (0, 0)→(0, 1)→(1, 1)→...→(3, 3)
- 마지막 스텝 (3, 3): cum_reward = 0
  - reward=-1, cum_reward = -1 + 1.0x0 = -1
  - reward=-1, cum_reward = -1 + 1.0x(-1) = -2
  - reward=-1, cum_reward = -1 + 1.0x(-1=2) = -3
  - ... (스텝 수가 많을수록 음수가 커짐)

> 이동 횟수가 많을수록(목표에 늦게 도달할수록) 누적 보상이 더 작은 음수가 된다.                 
> 따라서 가치가 낮은 상태 = 목표까지 멀리 돌아가는 상태이다.
